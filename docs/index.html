Streamline Azure IP Tracking & Automation

Azure Service Tags Tracker is a free, serverless tool that automatically monitors Azure Service Tag and IP range changes. It highlights weekly diffs, helps prevent firewall and policy breakages, and provides JSON output for automation and alerts.

Official Microsoft Data, Automatically Synced

‚ñº

The platform pulls the official Azure Service Tags JSON every Monday at 00:00 UTC via GitHub Actions. Each run produces current.json, differential reports, and timestamped history so you always know when an IP range was added or retired.

Built for Security & Networking Teams

‚ñº

Search for services like AzureStorage, regions such as East US, or specific IP addresses to easily identify range changes. Export change sets and sync results with firewall automation.

API & Automation Friendly

‚ñº

Every dataset lives on GitHub Pages under /data. Tools can fetch summary.json to discover available dates, compare /history/ snapshots for precise diffs, or read manifest.json to queue downloads. Sample scripts for PowerShell, Python, and more are provided.

What makes this different from Microsoft's raw file?

‚ñº

Microsoft publishes a single JSON, but this tracker layers on weekly comparisons, searchable UI, analytics, and long-term archives. You can quickly answer questions like "When did AzureCloud.eastus change?" without diffing files manually.

Can I monitor Azure IP changes without forking?

‚ñº

Yes. Point your scripts to https://eliaquimbrandao.github.io/azure-service-tags-tracker/data/ and query the JSON endpoints directly. No API keys, rate limits, or authentication flows required.
