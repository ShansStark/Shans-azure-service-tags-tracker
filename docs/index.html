How Azure IP Changes Are Tracked

The tracker continuously processes the official Microsoft Azure Service Tags dataset to generate weekly diffs and a searchable change history, enabling teams to quickly understand what changed and integrate updates into security and automation workflows.

üîÑ

Official Microsoft Data, Automatically Synced

The platform pulls the official Azure Service Tags dataset every Monday at 00:00 UTC via GitHub Actions. Each run generates a current snapshot, weekly diffs, and timestamped history so you always know when IP ranges are added or retired.

Built for Security & Networking Teams

Search by service, region, or IP address to quickly identify range changes and understand what changed and when. Detailed change sets can then be exported from the Change History for firewall and network automation.

API & Automation Friendly

‚ñº
